关于¶

目标¶

叉笔者的目标是培育和扶持, 通过公益诉讼的积极发展:

  • 通过 Travis CI, AppVeyor 和 GitHub Actions 持续集成测试

  • 在 GitHub 宣传开发活动

  • 定期发布到 Python Package Index

执照¶

像PIL, Pillow 开源软件PIL许可协议进行许可

为什么叉子?¶

PIL不兼容setuptools的.请参阅 this Image-SIG post 更详细的解释. 此外, PIL目前每两年(或更高版本)的发布计划是太疏, 以容纳数量众多, 而且问题频报.

什么PIL?¶

注解

此前Pillow2.0.0, 很少有图像的代码进行了更改.Pillow2.0.0添加Python 3的支持, 包括来自许多贡献者许多bug修复.

由于自上次发布PIL(1.1.7 2009年), 时间越久, 新PIL释放的可能性减小. 但是, 我们还没有听到官方 “PIL死了” 公告. 所以, 如果你仍然想支持PIL, 请 report issues here first, 然后使用 open corresponding Pillow tickets here.

请提供给第一票一个链接, 这样我们就可以跟踪这个问题(一个或多个)上游.
